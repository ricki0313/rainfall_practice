# day1 探索資料、PP01理解、-999.6處理
## PP01 欄位內容
1. 總列數: 469170
2. -999.1(儀器故障待修): 4464
3. -999.6(資料累積於後): 13715
4. -999.5(因故障而無資料): 189
5. None(未觀測而無資料): 1176

## -999.6 生成機制
- 雨水過濾斗，進入接水器，累積 0.5mm 後將水傾倒，產生脈衝信號，傳輸到紀錄器上，因此可看到資料是以 0.5 mm 為單位 
    - 來源: https://south.cwa.gov.tw/inner/meck1572422009QjcD
- 工程邏輯上，出現無線電訊號中斷，該時間點會標記為-999.6，並在恢復信號時間點將中斷期間的雨量觀測累加於後
    - 來源: https://w3fs.tainan.gov.tw/001/Upload/137/epaper/342/%E5%8F%B0%E7%81%A3%E5%9C%B0%E5%8D%80%E7%9F%AD%E5%BB%B6%E6%99%82%E5%BC%B7%E9%99%8D%E9%9B%A8%E4%BA%8B%E4%BB%B6%E6%B0%A3%E5%80%99%E7%89%B9%E6%80%A7%E5%88%86%E6%9E%90-%E8%BD%89%E8%BC%89%E5%9C%8B%E7%81%BD%E7%81%BD%E5%AE%B3%E9%98%B2%E6%95%91%E7%A7%91%E6%8A%80%E4%B8%AD%E5%BF%83%E7%81%BD%E5%AE%B3%E9%98%B2%E6%95%91%E9%9B%BB%E5%AD%90%E5%A0%B1%E7%AC%AC132.pdf
- 資料分析層面，使用｢資料累積於後(-999.6)」，而不是直接補上斷訊期間脈衝，推測回補時間定位並非 100% 可靠。我們也能看到 0 -> -999.6 -> 0的數據，在此情況卻不將-999.6 設定為 0，顯示其不確定性。

## 初步處理 -999.6
1. 不更動原數據
2. 新增 `PP01_cleaned`，將遺失值全部改為 na
3. 新增 `PP01_accumulated` 標註 -999.6
4. 新增 `PP01_caaumulated` 標註 -999.6 影響範圍
    - -999.6 -> 標示 True
    - -999.6 且 下一列 不為 0 -> 下一列標示 True
    - -999.6 且 下一列 為 0 -> 下一列標示 False
- 來源: WMO Guide to Climatological Practices (WMO-No.100) 2.4
